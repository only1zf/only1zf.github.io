Rust 极致编译加速手册（AI 开发版）

目标：本地秒级编译、运行满血性能、跨 worktree 复用产物、生产镜像最小。

本手册与项目内 .cargo/config.toml 与 Dockerfile 严格对齐，作为只读说明；改配置请直接改源文件。

1. 环境准备​

工具用途说明
sccache跨 worktree 共享 rustc 编译产物新 worktree clean build 52s → 22s
moldELF 极速链接器仅 Linux 生效，macOS Mach-O 无 mold 变体
cargo-sweep清理冗余 target/AI 多项目场景必备
nightly toolchainrustfmt 增强（import 重排）cargo +nightly fmt
cargo install sccache cargo-sweep
rustup toolchain install nightly --component rustfmt
# Linux only
sudo apt install -y mold

2. 项目配置（.cargo/config.toml）​

2.1 sccache 与 incremental 互斥​

cargo 增量编译产物 non-cacheable，sccache 见到 incremental flag 直接 bypass。两者择一：

  • 跨 worktree 优先（默认）：全局关 incremental，sccache 全速。
  • 主仓单 worktree 高频迭代：临时 CARGO_INCREMENTAL=1 cargo build --release，shell 级覆盖即可，sccache 自动 bypass。
[build]
rustc-wrapper = "sccache"

# 强制关 incremental 让 sccache 可缓存（profile 默认值挡不住 cargo 传的 incremental flag）。
[env]
CARGO_INCREMENTAL = "0"

[registries.crates-io]
protocol = "sparse"

2.2 链接器按 target 隔离​

# homebrew mold 只装 ld.mold(ELF)；macOS Mach-O 在 clang 侧拒绝 -fuse-ld=mold。
# 故 rustflags 必须挂在 Linux target 下，不能放 [build]。
[target.x86_64-unknown-linux-gnu]
rustflags = ["-C", "link-arg=-fuse-ld=mold"]

2.3 release profile：自身 0 / 依赖 3​

[profile.release]
# 本地：自身代码不优化，编译飞快。
# 生产：Dockerfile 里 ENV CARGO_PROFILE_RELEASE_OPT_LEVEL=3 覆盖此行。
opt-level = 0
debug = 0
panic = "abort"
lto = false
codegen-units = 256

# 第三方依赖永远满血，与 opt-level 无关：本地运行时也是生产速度。
[profile.release.package."*"]
opt-level = 3

codegen-units = 256 把 crate 拆为 256 个并行单元，本地多核 CPU 全速运转，以放弃跨单元内联换取编译时间；与 opt-level=0 协同——反正自身代码不优化，内联损失忽略。

3. 格式化（rustfmt.toml）​

nightly 才能重排 import：

unstable_features = true
imports_granularity = "Crate"
group_imports = "StdExternalCrate"
format_code_in_doc_comments = true

执行：cargo +nightly fmt

4. Docker 生产构建（实际 Dockerfile）​

四个关键决策：

  1. CARGO_PROFILE_RELEASE_OPT_LEVEL=3 覆盖本地 opt-level=0，生产强制满血。
  2. RUSTC_WRAPPER= 清空 覆盖仓库 .cargo/config.toml 的 rustc-wrapper=sccache——镜像未装 sccache，留着 cargo 直接报错。
  3. CARGO_TARGET_DIR=/app/target 必须显式指到挂载路径，否则 cache mount 是摆设（cargo install 默认走临时 target）。
  4. distroless 多阶段 最终镜像无 shell/curl/wget，健康检查走内建 health-check 子命令做 TCP 自探。
FROM docker.io/rust:1.95-trixie AS builder
WORKDIR /app
COPY . .

ENV CARGO_PROFILE_RELEASE_OPT_LEVEL=3 \
    CARGO_TARGET_DIR=/app/target \
    RUSTC_WRAPPER=

# BuildKit 三挂载：registry/git 缓存依赖源码，target 缓存编译产物。
# cache mount 不入镜像层，故 --root /out 把 binary 落到持久层。
RUN --mount=type=cache,target=/usr/local/cargo/registry \
    --mount=type=cache,target=/usr/local/cargo/git \
    --mount=type=cache,target=/app/target \
    cargo install --path frameworks --root /out

# 归集到 /rootfs 让运行阶段单条 COPY 落一层。
RUN set -eux; \
    mkdir -p /rootfs/app/web; \
    cp /out/bin/* /rootfs/app/entrypoint; \
    cp config.yaml /rootfs/app/config.yaml; \
    cp -r migrations /rootfs/app/migrations; \
    cp web/*.{html,js,css,ico} /rootfs/app/web/

FROM gcr.io/distroless/cc-debian13:nonroot
WORKDIR /app
COPY --from=builder --chown=nonroot:nonroot /rootfs/app /app
EXPOSE 61188

HEALTHCHECK --interval=30s --timeout=5s --start-period=15s --retries=3 \
    CMD ["/app/entrypoint", "--config", "/app/config.yaml", "health-check"]

ENTRYPOINT ["/app/entrypoint", "--config", "/app/config.yaml"]
CMD ["serve"]

5. 磁盘空间管理​

target/ 是 AI 多项目场景的吞磁盘黑洞，sccache 自带 LRU 自动管理全局缓存，本地 target/ 用 cargo-sweep：

cargo sweep -r --time 30      # 清 30 天未编译的项目产物
cargo sweep -r --installed    # 清非当前 rustc 版本生成的旧产物

6. 日常开发流​

# 编译/测试一律 --release（项目 CLAUDE.md 硬约束）
cargo build --release
cargo nextest run --release --workspace
cargo clippy --release --all-targets --workspace

# 主仓单 worktree 高频迭代（sccache bypass 走增量）
CARGO_INCREMENTAL=1 cargo build --release

# 格式化
cargo +nightly fmt
